# STORY-011: Getting started guide — Acceptance Criteria

## AC-001: Approachable for new users

**Given** someone has never used a context engineering framework
**When** they read the getting started guide
**Then** they understand what Lit SDLC is and why it exists
**And** they can follow the steps to install it in their project
**And** they know how to start their first session (/bootstrap → /start-session → /shape-spec)

## AC-002: Links to deeper content

**Given** the getting started guide introduces a concept (e.g., standards, specs, skills)
**Then** it links to the relevant deep-dive documentation
**And** does not attempt to explain everything inline
